What types of queries do i want? (filter by...)
- query by your champion 
	- this might be a prerequisite for other filters.
- query by champion on enemy team
	- might have to code lane prediction , but idk
	- should be able to match up mid zed vs mid veigar, as opposed to mid veigar vs zed jg
        - needs a POST request in the match history (lane correction)
    - in details, should be able to query an ADDITIONAL champion (up to 4)
- query by champion on your team
    - should exclude the champion you play, only other 4 slots
- query by rune you took
	- major rune or minor rune? (minor rune might be annoying idk)
- query by item you built
	- robustness- what if you sold all items end of game and bought 6 tears
- query by dragons
	- we got -'soul' OR <choice of soul>
	- they got - 'soul' OR <choice of soul>
	- show winrate
- query by rift herald
	- same as dragons
- query by time
	- game ended 20 - 25 minutes
	- game ended > 30 minutes
	- game ended < 16 minutes 20 seconds
- query by patch - be able to specify from what patch to what patch. (whats your winrate with patch)
	- patch 12.0 - 12.0
	- patch 12.1 - now
- query by elo
	- eliminate smurf queue outliers and take the average elo
- by summoner name on team/enemy team (make 2 different settings probably)
	- should conjure a winrate with, winrate against.
	- should show what champions they picked and winrate
	- robustness - what if a summoner changes his name? 
- should be able to manually delete a game from the CURRENT search (so it doesnt affect the stats)
	- maybe if you click something, it grays out and the stats get updated.
	- making a post request to actually delete something from a search requires more infrastructure, i dont wanna do it

Riot stores match data on match-v5. How far back can I go?
you can set 'count' to get 100 games.
you can set 'start' to start anywhere from 1-1000.
you can get up to the last 1000 games or up to 2 years whichever is less.

For my project, I want to store previous data too. Matches from seasons ago. Riot will eventually delete this data from their servers.
Therefore, it is necessary to make a list of WHAT needs to be stored on the backend to properly query matches from seasons ago.
The problem here is that I'm not familiar enough with the API to know this. But i will make a list of potential problems I need to cover.

Rune reworks? item overhauls? does data dragon have previous items? Champion changes? 
- this means that I cannot query things every time
- the solution is the keep an inhouse list of itemIDs, mapping them to their actual names.
    - if stormrazor gets removed next season, keep itemID 1234 ---> "stormrazor" + icon. that way, last season games can still be queried.
